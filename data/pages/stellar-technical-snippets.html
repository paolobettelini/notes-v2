Snippets

Snippets are the most basic units of contents. A snippet may be just an image, some text, a mathematical expression, a mathematical theorem or proof, an animation or an interactive animation.

Snippet structure

Any snippet is a folder, named as the identifier of the snippet, containing the content to display. The main snippet file is also named as its identifier and has one of the supported extensions.

Supported extensions are: html, pdf, png (TODO).
Examples:
  • snippets/
    • some-snippet-id-1
      • some-snippet-id-1.html
    • some-snippet-id-2
      • some-snippet-id-2.pdf

Complementary files

A folder for a snippet may multiple multiple files. The main file may request the contents of the complementary files.
  • snippets/
    • some-snippet/
      • some-snippet.html
      • complementary1.html
      • complementary2.html
The main file some-snippet.html can request the file complementary1.html by the the following uri: 
/snippet/some-snippet/complementary1
 If a complementary file wants to request another complementary file (E.g. complementary2), it is sufficient to request the URI 
./complementary2
.
